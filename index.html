Ultrex Studio

AI Shorts Engine

Ultrex came as an idea last year when I realized I was spending more time consuming content on YouTube than creating it. I decided I was going to learn video editing and start creating shorts myself.

After learning a bit and creating some shorts, I quickly realized the process was not sustainable because of how much time it consumed. I knew there had to be a better way.

That was where Ultrex came from. I initially built it to accept short ideas and use AI to generate images and audio, then use MoviePy to compile everything into a video. It worked, but generation time was too slow.

I researched better alternatives and moved to FFmpeg for optimized media processing, which reduced generation time drastically and made it possible to create a short in about 30 seconds or less.

Then I noticed another bottleneck - getting good scripts. I found that spending time prompting and refining outputs manually still slowed the process down.

So I pushed it further and created an AI agent workflow inside Ultrex that handles creating, reviewing, improving scripts, processing media, and even uploading directly to YouTube.

The result became a system that can go from an idea for a short to a published YouTube video in under 3 minutes. I tested it personally and within three days generated about 400 views and 1.4 hours of watch time.

I haven't deployed it publicly yet because of infrastructure costs involved in hosting video processing pipelines, and currently I want to perfect the experience before making it public.

Tools used: FastAPI, Uvicorn, APScheduler, Groq API, LangGraph, Pollinations AI, Edge-TTS, FFmpeg, Turso, Google OAuth, YouTube Data API, Pydantic, Mutagen, python-dotenv.

Arziki

Inventory forecasting for SMEs

Arziki started as a competition idea after a friend reached out and asked if I would be interested in joining the team.

The goal was to build an inventory intelligence system for Small and Medium Enterprises, with our focus being supermarkets.

My responsibility was building the machine learning pipeline - ingesting raw inventory data, transforming it into usable inputs, forecasting future stock levels, detecting possible stockouts, and generating reports and insights.

I still plan to continue developing it and eventually bring it fully to life.

SmartMeter

Household electricity forecasting

SmartMeter was a final year project I built for a Computer Science student.

The goal was to create an effective machine learning solution that could forecast household electricity consumption while still being lightweight enough to run on lower-cost devices.

I experimented with five different models, compared their performance, and exposed all the analysis directly inside the web application.

To make interaction easier, I also integrated an AI assistant that could answer basic questions about the project and the student.

What I enjoyed most while building this was balancing model performance with practicality and deployment considerations.

Tools used: Python, pandas, scikit-learn, statsmodels, TensorFlow, Streamlit, Groq, joblib, python-dotenv.

Lassa Fever Informatics

Outbreak analytics for Ondo State

Lassa Fever Informatics was another final year project I built for an Information and Communications Engineering student.

The goal was to create a system that could visualize disease trends, identify hotspots, and predict future Lassa fever outbreaks in Ondo State.

I built the analytics pipeline, visualizations, and prediction components.

Because the project used confidential data, I only made the web application public and did not upload notebooks or datasets.

This project gave me experience working with sensitive data and building responsibly around real-world constraints.

Tools used: Python, pandas, numpy, scikit-learn, Streamlit, Plotly, Folium, geopy, Jupyter Notebook.
